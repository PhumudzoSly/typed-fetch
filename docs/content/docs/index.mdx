# Welcome to typed-fetch

<Callout type="info" title="What is typed-fetch?">
**Status-aware `fetch` wrapper that learns response shapes and generates TypeScript types from real traffic.**

typed-fetch eliminates manual type definitions for HTTP APIs while maintaining strict privacy. It observes your real network traffic, infers response shapes, and generates accurate TypeScript types—all without storing raw data.
</Callout>

### Basic Example

```ts
import { typedFetch } from '@phumudzo/typed-fetch';

const result = await typedFetch(
  'https://api.example.com/user/123',
  { method: 'GET' },
  { endpointKey: 'GET /user/:id' }
);

if (result.status === 200) {
  console.log(result.data); // ✨ fully typed!
}
```

### Generate Types

Run your app to collect observations, then:

```bash
npx typed-fetch generate
```

Generated `typed-fetch.d.ts` augments the global type registry with your API shapes.

### Type-Safe Results

```ts
if (result.status === 200) {
  // result.data is now fully typed based on observed responses
  const name = result.data.name;
  const email = result.data.email;
} else if (result.status === 404) {
  console.error('User not found');
} else if (result.status === 0) {
  // Network error
  console.error('Network error:', result.error?.message);
}
```

## Key Features Explained

### Never Throws Exceptions

Unlike native `fetch`, typed-fetch returns all errors as result objects:

```ts
const result = await typedFetch(...);

// No try-catch needed!
if (result.status === 0) {
  // Network error
  handleNetworkError(result.error);
} else if (!result.ok) {
  // HTTP error
  handleHTTPError(result.status, result.data);
} else {
  // Success
  processData(result.data);
}
```

### Privacy by Default

- **No raw values stored**: Only response structure is recorded
- **Strict privacy mode**: Raw paths not saved (default)
- **Field filtering**: Sensitive fields automatically ignored
- **Local storage**: All data stays in your project

### Status-Based Type Narrowing

TypeScript automatically narrows types based on status codes:

```ts
const result = await typedFetch(...);

if (result.status === 200) {
  // TypeScript knows result.data shape for 200
  result.data.property; // ✅ typed
}

if (result.status === 404) {
  // TypeScript knows result.data shape for 404
  result.data?.message; // ✅ typed
}
```
